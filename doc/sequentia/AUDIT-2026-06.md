# Sequentia — Security & Spec-Conformance Audit (2026-06)

Method: a multi-agent audit — extract a testable spec from the white papers (`doc/sequentia/*`),
audit each subsystem + UI against it in parallel, then adversarially verify every finding
(default = refuted unless demonstrated). 48 agents; severities below are the **verified** ones
(several were revised down on verification). 0 critical, ~12 high, ~14 medium confirmed.

Scope: node consensus/fee subsystem, asset registry service, price-server sidecar, web wallet,
explorer, node GUI. **Not exercised:** Bitcoin anchoring / reorg correctness (see honesty notes).

---

## HIGH

- **HIGH-1 — Wallet reissue/burn field labeled "(atoms)" but parsed as decimal whole-units (1e8× over-burn).** `sequentia-web-wallet/index.html:234` label vs `:1011`/`:1017` `parseAtoms(value, m.precision)`. Irreversible. Fix: relabel `Amount`, placeholder `0.0`, keep decimal parse (consistent with issuance/send/stake); show resolved atom count in the confirm modal.
- **HIGH-2 — Every Sequentia money path shows a blind "Ready to sign" review.** `reviewBroadcast` (`:888-904`) relies on `psetDetails`, which throws on Sequentia's explicit (default) outputs → the catch shows nothing for send/issue/reissue/burn/stake/RBF/CPFP. Fix: render structured rows (address + amount + ticker + ≈ ref + fee) from `collectRows()`/dialog inputs, not `psetDetails`. **Keystone fix** — would surface HIGH-1/HIGH-9.
- **HIGH-3 — Asset registry fetched over unauthenticated plaintext HTTP, feeds spend-asset resolution.** `src/assetregistry.cpp:58-137` (http only), sink `wallet/rpc/spend.cpp` via `GetAssetFromString`. MITM/DNS hijack can repoint ticker→asset-id. (Testnet narrows it: 6 demo tickers are pre-seeded + protected; any *other* ticker is unauthenticated; mainnet has no pre-seed.) Fix: sign the index with a chainparams-pinned key and verify before `MergeGlobalAssetDir`, or re-derive `contract_hash` against the chain.
- **HIGH-4 — No ticker uniqueness in the registry (squatting / non-deterministic bind).** `sequentia-registry/server.js:157-191`. (Downgraded from the filed claim — bind is insertion-order, not hex-order; demo tickers protected.) Fix: enforce ticker/name uniqueness in `register()`+`loadSeed`; node-side drop duplicate tickers.
- **HIGH-5 — Live registry set is entirely unverified, and the minimal index drops the verified flag.** All 8 live assets `verified:false`; node can't tell verified from legacy. Fix: add a verified flag to the minimal-index tuple; node skips/badges non-verified; reject all-zeros issuer pubkey.
- **HIGH-6 — Price-server config UI `/save` is unauthenticated + no CSRF.** `contrib/price-server/price_server.py:489-499`. A drive-by POST rewrites admission rules → fans out to the committee via `setdynamicfeerates`. Fix: Origin/Referer check + auth + CSRF token; refuse non-loopback bind without an explicit flag.
- **HIGH-7 — Sidecar never clamps rates to int64; one bad source drops the whole node whitelist.** `price_server.py:215,326,345` → node `get_int64()` throws on overflow → whole batch rejected. Fix: clamp `0 < rate ≤ ~1e15`; drop+log offenders instead of poisoning the batch.
- **HIGH-8 — Broadcast forwarder returns empty body (not the txid) on benign re-broadcast.** `sequentia-explorer/serve-public.js:78-85` (line 83): already-in-chain → empty 200 → client navigates to `/tx/` (empty). Fix: recover the txid via `getrawtransaction`/`decoderawtransaction`; never `send(out)` when empty. (Do NOT parse the txid from the error string.)
- **HIGH-9 — Non-native fee asset rate silently falls back to 1:1 → catastrophic OVERPAYMENT.** `sequentia-web-wallet/index.html:870,1120,1154,1191`; catch `:388` clobbers `feeRates={}`. (Mechanism corrected: priced-but-stale → builds too few atoms at 1:1, node values at real rate → user overpays; unlisted → rejected.) Fix: hard-error when `feeRates[asset]` absent; keep last-good `/feerates`; gate unpriced assets in selectors.
- **HIGH-10 — Node-GUI fee-policy "Add / update" wipes the entire fee whitelist.** `src/qt/feepolicydialog.cpp:141-153,202-221` still reads the removed `origin` field → `currentManualRates()` empty → edit one asset deletes all others. Fix: build the merge base from the FULL current policy; enable Remove; derive "Managed by" without the removed field.
- **HIGH-11 — Node-GUI fee-policy "Launch price server" uses a hardcoded `/home/aejkohl/...` dev path.** `src/qt/feepolicydialog.cpp:279,282` — dead for every user. Fix: resolve relative to `applicationDirPath()` (or call the shared `bitcoingui.cpp::launchPriceServer`); never ship a `/home/<user>` path.
- **HIGH-12 — Price-server launcher config seeding is broken.** `src/qt/bitcoingui.cpp:995-1010` runs `gen-price-config.py` and writes its stdout, but that script writes to `/root/price-demo/config.json` and prints only a summary → invalid/missing config → sidecar crashes. Fix: copy a bundled `config.example.json` to `<appdata>/price-server.json`; verify the sidecar binds before claiming success.

## MEDIUM (abridged)

- **MED-1 / MED-11 — fee-market docs vs code drift** (single-whitelist model; rate 0 = refuse not reject; no `-dynfeeratemaxage`; sidecar owns the whole whitelist). Rewrite `doc/sequentia/02-open-fee-market.md` §2/§5 + the price_server docstring.
- **MED-2 — registry domain proof is a weak substring match, not bound to issuer_pubkey; SSRF** (`server.js:127-138,94-107`). Require exact body + text/plain; bind issuer_pubkey via signature; private-IP guard.
- **MED-3 (Low) — contract binding trusts electrs's contract_hash without re-deriving the asset id.**
- **MED-4 — wallet ⇄ swap write-back uses fixed 8dp → un-sendable for sub-8-precision assets** (latent; all live assets are 8dp; fails closed). Round to per-asset precision.
- **MED-5 — owned-but-unlabeled asset defaults to precision 0 → send mis-parse** (`index.html:359`). Discover on-chain precision or block with a warning.
- **MED-6 — sub-minimum stake (<40,000 tSEQ) accepted with no warning** in the web wallet (`:1236-1250`); the desktop wallet already guards this. Mirror the guard.
- **MED-7 — shared tb1 across chains: no chain-disambiguation in the Sequentia review.** Add a "Network: Sequentia (testnet)" row (depends on HIGH-2).
- **MED-8 / MED-9 / MED-10 — price-server admission gate bypasses** (max_change_factor baseline reset; zero-criteria admission; reference-asset SPOF clears the whole whitelist on a momentary outage).
- **MED-12 — explorer forwarder can strand a tx the explorer node accepts but the producer rejects, retrying one producer forever.** Succeed if either send returns a txid; forward to multiple producers; bound retries.
- **MED-13 — explorer faucet per-IP limit forgeable via X-Forwarded-For; `faucetSeen` grows unbounded.** `trust proxy` + `req.ip`; validate addresses; evict stale entries.
- **MED-14 — price-server launcher reports success even when the sidecar immediately dies** (`startDetached`). Poll the UI port; pick a free port instead of hardcoded 8089.

## PER-UI VERDICT (all AT-RISK)
- **Web wallet** — not safe to ship as-is: blind review (HIGH-2), 1e8× over-burn label (HIGH-1), 1:1 fee overpayment (HIGH-9).
- **Explorer** — re-broadcast returns empty txid (HIGH-8), cross-acceptance stranding (MED-12), forgeable faucet (MED-13).
- **Registry** — highest-priority: unauthenticated transport feeding spends (HIGH-3), no ticker uniqueness (HIGH-4), unverified live set (HIGH-5).
- **Price-server** — CSRF-able admin endpoint (HIGH-6), unbounded rates (HIGH-7), gate bypasses + SPOF (MED-8/9/10).
- **Node GUI** — fee-policy wipes the whitelist (HIGH-10); launchers broken (HIGH-11/12, MED-14).

## OPERATOR / RUNTIME NOTES (from live testing — not caught by the static audit)

- **RUN-1 — No automatic peer discovery; node stuck at height 5499.** `src/chainparams.cpp:489` does `vSeeds.clear()` for the Sequentia testnet (no DNS seeds, no fixed seeds), so the node only ever connects to `-addnode` peers. Needs: a seed mechanism (DNS or `chainparams_seed` fixed seeds once stable nodes exist) and/or addr-gossip from the gateway; plus a live diagnosis of why sync stalls at 5499 (gateway peer height vs block production).
- **RUN-2 — "Check BTC balance" button → "address is not valid"; BTC should auto-load.** `src/qt/overviewpage.cpp:207` button → `getbtcbalance` RPC (`src/wallet/rpc/spend.cpp:184`) erroring. Design: BTC (parent-chain) balance should load automatically like Sequentia assets, not behind a button.
- **RUN-3 — Send-tab amount dropdown is the precision picker (tSEQ/mtSEQ/µtSEQ/Sat), not an asset selector.** `BitcoinAmountField` (`src/qt/bitcoinamountfield.h:88` `QComboBox *unit`) falls back to `BitcoinUnits` precision when the wallet has no/one asset type (`setAllowedAssets`/`updateAssetTypes`). Should always be a whole-unit ASSET selector (the unit-picker removal missed this field).
- **RUN-4 — Price Server menu → "Could not find price_server.py".** Corroborates HIGH-11/12/MED-14: the launcher path resolution + config seeding are broken for non-dev binaries.
